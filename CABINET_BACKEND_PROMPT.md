## Контекст проєкту

Це освітня школа для дітей **Firefly School** (Вінниця). У нас вже є робочий статичний лендінг
у цій папці (`site/`): `index.html`, `styles.css`, `script.js`, `assets/`. Він на GitHub.

Ми вже зробили **візуальний прототип особистого кабінету** — файл `cabinet.html` у цій же папці.
Це повністю клікабельний демо (на чистому HTML/CSS/JS, дані захардкоджені), у фірмовому стилі:
- шрифти **Comfortaa** (заголовки) + **Manrope** (текст)
- кольори: золото `#FFC800`, помаранчевий `#E8821E`, коричневі тони `#1B0F09 / #2A1A12 / #3A251A`
- логотип: `assets/logo-clean.png`
- кнопки `.btn--primary` / `.btn--ghost`, радіуси, тіні — як у `styles.css`

**Завдання:** перетворити цей демо-кабінет на справжній застосунок із реальним входом,
базою даних і онлайн-оплатою. Головні користувачі кабінету — **діти** (дружній, простий UI),
оплатами керують батьки.

## Що має вміти кабінет (функції з прототипу)

1. **Вхід / реєстрація** учня (зараз демо — приймає будь-що)
2. **Головна** — найближчий урок, статистика тижня, список уроків
3. **Календар уроків** (місяць + розклад тижня)
4. **Запис на урок** і **скасування** уроку (вибір вільних слотів)
5. **Домашні завдання** — список, відмітка «виконано», +бали
6. **Бали та бейджі** (гейміфікація, рівні, прогрес)
7. **Абонемент / оплата** — купити місячний абонемент, разовий урок, безліміт; скасувати абонемент; історія платежів

---

## Рекомендований стек (і чому)

- **Frontend:** Next.js (React, App Router) + TypeScript. Перенести дизайн із `cabinet.html` у компоненти, зберегти ті самі CSS-токени.
- **Auth + база даних:** **Supabase** (Postgres + Auth + Row Level Security). Безкоштовний тариф, швидко стартувати.
- **Хостинг:** **Vercel** (під'єднується до того ж GitHub-репо, автодеплой).
- **Платежі (Україна):** **WayForPay** або **Fondy** (підтримують українські ФОП, рекурентні платежі для абонементів). ⚠️ Stripe виплати для українських компаній не підтримує — не використовувати.

> Перед інтеграцією платежів перевір актуальні умови та документацію обраного провайдера.

## Модель даних (чернетка таблиць Supabase)

- **profiles** — учень: id (= auth user), імʼя, аватар, роль (`child` / `parent`), parent_id, рівень, бали
- **teachers** — викладачі: id, імʼя, напрямок
- **subjects** — напрямки: id, назва, емодзі/іконка
- **lesson_slots** — доступні слоти: id, subject_id, teacher_id, дата-час, місткість, статус
- **bookings** — записи: id, student_id, slot_id, статус (`booked` / `attended` / `cancelled`)
- **homework** — завдання: id, student_id, subject_id, текст, дедлайн, done (bool)
- **badges** + **student_badges** — бейджі та які отримав учень
- **plans** — тарифи: id, назва, ціна, к-ть уроків, період
- **subscriptions** — абонементи: id, student_id, plan_id, статус, активний_до, уроків_залишилось
- **payments** — платежі: id, student_id, plan_id, сума, статус, provider_ref, дата

Налаштувати **RLS**: учень/батьки бачать лише свої дані.

## Поетапний план (іди по кроках, після кожного — коміт)

1. **Підготовка:** створити проєкт Supabase і Vercel; додати ключі в `.env.local`; не комітити секрети.
2. **Скаффолд Next.js** у підпапці (напр. `app/`), не ламаючи наявний статичний лендінг.
3. **Дизайн-система:** перенести CSS-токени та компоненти кнопок/карток із `cabinet.html`.
4. **Auth:** Supabase Auth (email+пароль), сторінки входу/реєстрації, захищені маршрути.
5. **Схема БД:** SQL-міграції для таблиць вище + RLS + сіди (демо-дані для тесту).
6. **Екрани:** Головна → Календар → Запис/Скасування → Завдання → Бали → Абонемент (на реальних даних).
7. **Платежі:** інтеграція WayForPay/Fondy, вебхук підтвердження оплати → оновлення `subscriptions`.
8. **Деплой** на Vercel, тест на тестовому акаунті учня.

## Важливі обмеження

- UI **українською**, дружній до дітей, зберегти фірмовий стиль (золото/коричневий, Comfortaa/Manrope, логотип-світлячок).
- Наявний лендінг (`index.html` тощо) має лишитися робочим; кнопка «Вхід» у шапці вже веде на кабінет.
- Не зберігати секрети/ключі в git. Дотримуватись безпеки (RLS, валідація, перевірка вебхуків).
- Використовувати `cabinet.html` як **візуальний референс** макета й текстів.
